Description

Mise en place d'un hyperviseur

Ce projet consiste en la configuration d'un serveur de virtualisation Proxmox hébergeant plusieurs machines virtuelles à des fins de test, de développement et de production simulée.

L'objectif principal était de comprendre et maîtriser la gestion des ressources partagées (CPU, RAM, stockage), la haute disponibilité et les mécanismes de sauvegarde via les snapshots.

Réalisations

Ce qui a été mis en place

  • Installation et configuration de Proxmox VE sur machine physique dédiée
  • Déploiement de 4 VMs : Windows Server 2022, Debian 12, Ubuntu Server, pfSense
  • Gestion des pools de stockage (LVM, ZFS) et allocation des ressources
  • Automatisation des snapshots quotidiens avec planification cron
  • Configuration du réseau virtuel (bridges, bonding)
  • Migration à chaud d'une VM entre deux nœuds (simulation cluster)
Interface Proxmox — Gestion des VMs

Points clés

Apprentissages techniques

La gestion des ressources dans un environnement virtualisé requiert une planification rigoureuse pour éviter la surallocation et maintenir des performances optimales pour chaque VM.

  • Compréhension approfondie du fonctionnement des hyperviseurs de type 1
  • Maîtrise des formats de disques virtuels (qcow2, raw, vmdk)
  • Gestion des sauvegardes et plans de reprise d'activité (PRA)
  • Monitoring des ressources via l'interface web Proxmox

Technologies utilisées

Résultats

Bilan

Le serveur de virtualisation est opérationnel et héberge en permanence plusieurs services de test. Les snapshots automatisés garantissent un retour arrière rapide en cas d'incident. Ce projet a considérablement renforcé mes compétences en administration système.
