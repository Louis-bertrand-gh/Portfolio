Description

Automatisation du déploiement

Ce projet vise à automatiser le déploiement et la configuration de serveurs Linux via des playbooks Ansible. L'approche Infrastructure as Code (IaC) permet de garantir des environnements reproductibles et de réduire les erreurs humaines liées aux configurations manuelles.

J'ai développé un ensemble de rôles Ansible couvrant l'installation de services web, la sécurisation de serveurs et le déploiement d'applications.

Réalisations

Playbooks développés

  • Playbook de hardening SSH (désactivation root, clés publiques, bannière)
  • Déploiement automatisé d'un serveur LAMP (Linux, Apache, MySQL, PHP)
  • Rôle de configuration du pare-feu UFW avec règles personnalisées
  • Automatisation des mises à jour de sécurité (unattended-upgrades)
  • Gestion des utilisateurs et droits sudo via inventaire Ansible
  • Déploiement d'un serveur de monitoring Grafana/Prometheus
Extrait d'un playbook Ansible YAML

Approche technique

Structure des rôles

L'utilisation de rôles Ansible permet de décomposer la configuration en modules réutilisables, maintenables et testables indépendamment.

  • Organisation en rôles modulaires (common, webserver, security, monitoring)
  • Utilisation des variables et des templates Jinja2 pour la généricité
  • Gestion des secrets avec Ansible Vault
  • Tests d'idempotence sur chaque playbook

Technologies utilisées

Ansible 2.15 YAML Jinja2 Debian 12 Apache2 UFW Ansible Vault SSH

Résultats

Bilan

Le temps de déploiement d'un nouveau serveur est passé de 45 minutes en configuration manuelle à moins de 5 minutes avec les playbooks. Tous les environnements sont désormais identiques et le risque d'oubli de configuration est éliminé.
